# Creative Techniques & Anti-Patterns

## Idea Generation Techniques

Use these techniques when generating ideas in Phase 2. Switch
techniques when exploration starts feeling repetitive rather than
stopping the ideation process.

### Inversion

Ask: what if the opposite of the obvious approach were taken?
If the obvious solution centralizes, what if it were distributed?
If the obvious approach adds features, what if features were removed?

### Analogy

Draw from genuinely unrelated domains. Other tech products do NOT
count as cross-domain. Think:

- **Biology**: How do ecosystems, evolution, or immune systems
  approach this kind of problem?
- **Cooking**: How does a chef think about mise en place, timing,
  or flavor pairing?
- **Sports**: How do teams coordinate, adapt mid-game, or train?
- **Urban planning**: How do cities manage flow, zoning, or growth?
- **Music**: How do composers build tension, create harmony, or
  structure a piece?
- **Filmmaking**: How do directors handle pacing, perspective, or
  audience expectations?
- **Architecture**: How do buildings balance form, function, and
  constraints?
- **Gaming**: How do game designers create engagement loops,
  progression, or emergent behavior?

Ask: "How would a [chef/architect/game designer] approach this?"

### Exaggeration

Take a feature to its extreme. What if it were 100x bigger, 100x
faster, 100x simpler? What becomes possible at the extreme that
isn't possible at normal scale? What new problems emerge?

### Subtraction

Remove the most obvious feature. What's left? Is that more
interesting? Sometimes the constraint of removing the "essential"
feature reveals what's truly essential.

### Combination

Take two unrelated ideas from the current session and combine them.
What emerges? Emergent properties often arise from unexpected
combinations that neither idea has alone.

### Perspective Shift

View the idea through different lenses:
- How would a first-time user see this?
- How would a power user see this?
- How would someone from a different culture see this?
- How would a competitor see this?
- How would someone who hates this category of product see this?

## Anti-Pattern: "I Have Enough To Start Building"

No. If fewer than 4 rounds of back-and-forth exploration have
happened, there isn't enough. If fewer than 5 original feature
ideas have been generated unprompted, the creative contribution
is insufficient. If what's explicitly out of scope hasn't been
defined, the scope is unbounded. Stay in ideation until the vision
is genuinely sharp.

## Red Flags - Leaving Ideation Too Early

If any of these are true, stay in ideation and keep exploring:

- Thinking about what framework to use
- Wanting to propose a database schema or API design
- Mentally structuring code or file layouts
- No idea in this session has been surprising
- The user hasn't pushed back on anything suggested
- Questions have been asked but ideas haven't been generated
- "What if..." hasn't been said at least three times
- The scope discussion took fewer than 2 exchanges

## Common Rationalizations for Cutting Ideation Short

| Excuse | Reality |
|--------|---------|
| "The idea is clear enough" | If 3 things explicitly out of scope can't be named, it's not clear |
| "We're going in circles" | Circles surface assumptions. One more round. |
| "The user seems ready to build" | The job is to ensure the idea is sharp, not to rush to code |
| "Enough questions have been asked" | Have enough ideas been GENERATED? Questions alone aren't ideation |
| "This is getting repetitive" | Switch techniques (inversion, analogy, subtraction) instead of stopping |
| "Simple ideas don't need deep exploration" | Simple ideas with unexamined assumptions cause the most wasted work |
